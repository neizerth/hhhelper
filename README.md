# HH Helper

Браузерное расширение для автозаполнения формы резюме на hh.ru по текстовому описанию.

## Технологии

- **Vite** + **vite-plugin-web-extension** — сборка расширения (Chrome MV3 / Firefox MV2 из одной кодовой базы, `src/manifest.json`)
- **Svelte 5** (runes) + **SMUI** — UI попапа
- **TypeScript**
- **webextension-polyfill** — кроссбраузерный `browser.*` API
- **Vitest** + **@testing-library/svelte** — тесты
- **Biome** — линт/форматирование
- **Lefthook** — git-хуки

## Архитектура (Feature-Sliced Design)

```
src/
  background/        service worker (live reload в dev)
  content.ts          точка входа content script
  background.ts        точка входа background
  slices/
    app/                сборка попапа (UI, storage последнего введённого текста)
    modules/
      browser/           отправка контента в активную вкладку (executeScript + fallback через runtime-сообщение)
      document/           сборка и запуск заполнения документа (fillDocument, реестр контролов)
      form/                домен формы:
        shared/            базовый Field, IFormControl, парсер текстового формата, низкоуровневый fillTextControlValue
        entities/          сложные контролы (месяц/регион/сфера деятельности/резюме к месту работы) — со своей логикой открытия попапов, поиска, ожидания асинхронного рендера
    shared/              общие утилиты (dom, string, async/waitFor)
```

Домен — **форма резюме hh.ru**: набор `IFormControl` (`Input`, `TextArea`, `Checkbox`, `Radio`, `MonthSelect`, `RegionSelect`, `CompanyScopeSelect`, `CVSelect`), каждый умеет по DOM-узлу вычислить `label` и применить `value`. `fillDocument` разбирает текст на пары `ключ: значение`, ищет по всем зарегистрированным контролам на странице подходящий `label` и вызывает `fill`.

Селекторы контролов стараются опираться на `role`/`aria-*`/`data-qa` атрибуты, а не на хэш-классы Magritte (они меняются от сборки к сборке).

## Как запустить

```bash
pnpm install
pnpm dev            # vite watch, автоперезагрузка контент-скрипта
pnpm dev:browser    # то же + автозапуск браузера с расширением
```

Собранное расширение — в `dist/`, загружается как unpacked extension в `chrome://extensions`.

```bash
pnpm build   # прод-сборка
pnpm check   # svelte-check
pnpm test    # vitest
```

## Релиз

```bash
pnpm release
```

Собирает прод-версию и упаковывает `dist/` в `releases/hhhelper-v<version>-chrome.zip` (версия берётся из `package.json`).

Автодеплой: при пуше git-тега вида `v1.2.3` GitHub Action ([.github/workflows/release.yml](.github/workflows/release.yml)) сам собирает и публикует архив в GitHub Releases (тот же `pnpm release` внутри). Порядок релиза: подними `version` в `package.json` → закоммить → `git tag v1.2.3 && git push origin v1.2.3`.

## Установка (без Chrome Web Store)

1. Скачай архив `hhhelper-v<version>-chrome.zip` со страницы [Releases](../../releases) (или собери сам: `pnpm install && pnpm release`).
2. Распакуй архив в отдельную папку.
3. Открой `chrome://extensions` (для Chrome/Edge/Brave — любой Chromium-браузер).
4. Включи **Режим разработчика** (переключатель в правом верхнем углу).
5. Нажми **Загрузить распакованное расширение** и укажи папку с распакованным архивом.
6. Иконка расширения появится на панели — можно закрепить.

Расширение работает только на `hh.ru`. При обновлении версии — повтори шаги 1–2 и нажми «Обновить» на карточке расширения в `chrome://extensions` (или удали старую версию и загрузи заново).

## Как пользоваться

1. Открой на hh.ru страницу редактирования резюме (новое резюме или блок опыта работы).
2. Открой попап расширения, вставь текст в формате `Ключ: значение` (по одной паре на строку) — тот же текст, что подписан у полей на странице.
3. Жми «Заполнить». Расширение найдёт на странице поля с такими подписями и подставит значения.

### Формат текста

```
Чем занимались?: Разработка фронтенда
Компания: ООО Ромашка
Должность или профессия: Frontend-разработчик
Сайт компании: https://example.com
Город или регион: Москва
Месяц: Март
Год: 2020
Работаю сейчас: нет
Месяц: Май
Год: 2021
Сфера деятельности компании: Разработка программного обеспечения, СМИ
Резюме: Frontend Разработчик (React), Преподаватель IT
```

- **`Ключ: значение`** — по одной строке. Значение — весь текст после первого `:`.
- **Булевы значения** (чекбоксы вроде «Работаю сейчас») распознаются автоматически: `да/нет`, `true/false`, `1/0`, `yes/no`, `y/n`.
- **Списки через запятую** — для мультиселектов (`Сфера деятельности компании`, `Резюме`): перечисли нужные пункты через запятую.
- **Повторяющийся ключ** (например, два поля «Месяц»/«Год» — начало и окончание работы) матчится по порядку появления на странице: первое вхождение ключа в тексте — первому полю на странице, второе — второму и т. д.
- **`Резюме: ...`** — служебный ключ, не привязан к подписи на странице. Управляет чекбоксами резюме в блоке «Резюме с этим местом работы»: перечисленные резюме отмечаются галочкой, остальные — снимаются. Если список резюме свёрнут («Развернуть»), расширение сначала раскроет его.
